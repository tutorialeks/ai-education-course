# Module 4: Learning Programming with AI

## Learning Objectives

By the end of this module, you will be able to:
- Use AI assistants effectively to learn programming concepts
- Debug code with AI help while building understanding
- Know when to use AI vs. when to struggle productively
- Build real projects with AI as a collaborative partner

## 4.1 From Zero to Coder with AI

### Choosing Your First Language

```
I want to learn programming. Help me choose my first language.

My goal: [CHOOSE ONE]
- Build websites
- Analyze data / work with spreadsheets
- Create mobile apps
- Automate repetitive tasks
- Get a job in tech
- Just understand how software works

My background:
- Technical experience: [none / some / experienced in other field]
- Math comfort level: [low / medium / high]
- Time available: [hours per week]

Recommend a language and explain:
1. Why it's good for my goal
2. What I'll be able to build after 3 months
3. Free resources to start learning
```

### The "Explain Like I'm 5" Approach

Don't settle for explanations you don't understand:

```
Explain [CONCEPT] as if I'm a complete beginner.

- No jargon without defining it first
- Use a real-world analogy
- Show the simplest possible code example
- Explain what each line does
```

Example concepts to learn this way:
- Variables
- Loops
- Functions
- Objects/Classes
- APIs
- Databases

### Building Mental Models

```
Help me build a mental model for [CONCEPT].

1. What is the core purpose? (one sentence)
2. How does it relate to things I already know?
3. What are the 2-3 key things to remember?
4. What's a common misconception beginners have?
5. Give me a minimal example that demonstrates the concept
```

### Step-by-Step Project Learning

The fastest way to learn is by building. Here's how to structure it:

```
I want to learn [LANGUAGE] by building a [PROJECT TYPE].

My current skill: [complete beginner / know basics / intermediate]

Create a step-by-step project guide:
1. Break it into 5-7 milestones
2. For each milestone:
   - What I'll build
   - New concepts I'll learn
   - Common pitfalls to avoid
3. Don't give me the code‚Äîgive me just enough guidance
   to figure it out myself
4. Tell me what to Google if I get stuck
```

## 4.2 Debugging and Problem-Solving

### The Right Way to Ask for Debugging Help

Bad approach: "My code doesn't work, fix it"

Good approach:
```
I have a bug in my [LANGUAGE] code.

What I'm trying to do: [GOAL]
What I expected: [EXPECTED BEHAVIOR]
What actually happens: [ACTUAL BEHAVIOR]
Error message (if any): [ERROR]

Here's the relevant code:
[CODE BLOCK]

What I've already tried:
- [ATTEMPT 1]
- [ATTEMPT 2]

Don't give me the fix directly. Instead:
1. Help me understand what's going wrong
2. Point me in the right direction
3. Let me try to fix it first
```

### Rubber Duck Debugging with AI

```
I'm stuck on a problem. Let me explain my code to you step by step.
Ask me clarifying questions as I go. Don't solve it for me‚Äîhelp me
realize where my thinking might be wrong.

My code is supposed to: [GOAL]

Let me walk through it line by line...
```

### Understanding Error Messages

```
Explain this error message in plain English:

[PASTE ERROR MESSAGE]

1. What does each part of the error mean?
2. What are the most common causes?
3. How do I find where the problem is in my code?
4. What should I check first?
```

### Learning from Your Mistakes

After fixing a bug:
```
I just fixed a bug where [DESCRIBE BUG].
The problem was [WHAT WAS WRONG].

Help me learn from this:
1. How could I have prevented this bug?
2. What pattern should I watch for in the future?
3. Are there tools/practices that catch this automatically?
```

## 4.3 Building Projects with AI Assistance

### The 80/20 Rule for AI in Coding

**Use AI for:**
- Understanding concepts and syntax
- Debugging confusing errors
- Boilerplate code (setup, config files)
- Learning new libraries/frameworks
- Code review and suggestions

**Don't use AI for:**
- Core logic you need to understand
- Every single line of code
- Skipping the struggle of learning
- Code you can't explain

### Productive Struggle

A 2025 study found that developers who felt 20% faster with AI sometimes took 19% longer overall because they didn't understand the code well enough to debug it.

**Balance:**
```
I'm building [PROJECT] and need to implement [FEATURE].

Before giving me code:
1. Explain the approach conceptually
2. Tell me what functions/methods I'll need
3. Point me to relevant documentation

Let me try implementing it first. I'll come back if stuck.
```

### Code Review by AI

After writing code, get feedback:

```
Review my code for:
1. Bugs or potential errors
2. Readability improvements
3. Performance issues
4. Security concerns
5. Whether it follows best practices for [LANGUAGE]

Be critical‚ÄîI want to learn, not just get validation.

[CODE BLOCK]
```

### Learning Design Patterns

```
Show me how to implement [PATTERN] in [LANGUAGE].

1. First explain when and why I'd use this pattern
2. Show a simple example
3. Show a real-world use case
4. What are common mistakes when implementing it?
5. How do I know if I'm overusing it?
```

## 4.4 AI Coding Tools (2026)

### Comparison of Tools

| Tool | Type | Best For | Price |
|------|------|----------|-------|
| **GitHub Copilot** | IDE plugin | Code completion | Free (students) / $10/mo |
| **Cursor** | IDE | Full AI development | Free tier / $20/mo |
| **Claude Code** | CLI/IDE | Complex debugging | Usage-based |
| **Windsurf** | IDE | Autonomous tasks | Free tier / Premium |
| **ChatGPT** | Chat | Explanations, learning | Free / $20/mo |
| **Replit AI** | Browser IDE | Beginners, no setup | Free tier / $25/mo |

### When to Use What

**Learning a new concept:**
- ChatGPT or Claude (conversational explanation)

**Writing code:**
- Cursor or Copilot (inline suggestions)

**Debugging complex issues:**
- Claude Code or ChatGPT (paste full context)

**Starting a new project:**
- Cursor Composer or Windsurf Cascade (scaffolding)

### Setting Up for Learning vs. Production

**For learning (maximizes understanding):**
- Use chat-based AI for explanations
- Write code yourself first, then compare
- Ask for feedback rather than solutions
- Disable autocomplete sometimes to test yourself

**For productivity (maximizes output):**
- Use inline completions (Copilot/Cursor)
- Accept suggestions for boilerplate
- Use AI for unfamiliar libraries
- Focus energy on architecture decisions


## Practical Exercises

### Exercise 1: Explain Back
Pick a concept you recently learned. Explain it to AI as if teaching someone else. Ask AI to point out any gaps or misconceptions in your explanation.

### Exercise 2: Bug Hunt
Intentionally introduce 3 bugs into working code. Practice debugging with AI, but don't let it give you the answer directly. Use the "point me in the right direction" approach.

### Exercise 3: Code Review
Write a solution to a coding problem. Submit it to AI for code review. Implement the suggestions and compare before/after.

### Exercise 4: Build Without AI
Spend one coding session without AI assistance. Note where you get stuck. Then use AI to fill those specific gaps. This reveals what you actually know vs. what you've been outsourcing.

---

## Key Takeaways

1. **AI accelerates, not replaces, learning** - You still need to understand the code
2. **Ask for guidance, not solutions** - The struggle is where learning happens
3. **Build projects, not just tutorials** - Application cements understanding
4. **Review your code** - AI code review catches blind spots
5. **Know when to unplug** - Test yourself without AI regularly
